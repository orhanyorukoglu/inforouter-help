# infoRouter Portals

Create customized entry points to infoRouter for different user groups or purposes.

---

## What Are Portals?

Portals are customized views of infoRouter that provide tailored experiences for specific user groups, departments, or external parties.

---

## Portal Types

| Type | Description |
|------|-------------|
| :material-home: **Main Portal** | Default entry point for all users |
| :material-account-group: **Department Portals** | Customized for specific departments |
| :material-account-multiple: **Customer Portals** | External access for clients |
| :material-handshake: **Partner Portals** | Vendor and partner collaboration |

---

## Portal Features

| Feature | Description |
|---------|-------------|
| :material-palette: **Custom branding** | Logos, colors, themes |
| :material-view-dashboard: **Custom layout** | Arrange components |
| :material-library: **Library visibility** | Show specific libraries |
| :material-link: **Quick links** | Direct access to common areas |
| :material-announcement: **Announcements** | Display news and updates |

---

## Benefits of Multiple Portals

| Benefit | Description |
|---------|-------------|
| :material-target: **Focused experience** | Users see only relevant content |
| :material-brush: **Brand consistency** | Match departmental or external branding |
| :material-security: **Simplified access** | Pre-configured for user groups |
| :material-navigation: **Easy navigation** | Reduced complexity |

---

## Common Portal Scenarios

| Scenario | Portal Setup |
|----------|--------------|
| **Customer access** | Client documents and support resources |
| **HR self-service** | Employee forms and policies |
| **Project workspace** | Team collaboration area |
| **Vendor portal** | Supplier documentation |
