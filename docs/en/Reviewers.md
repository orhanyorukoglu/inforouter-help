# Reviewers and Task Assignees

Reviewers are users assigned to examine, approve, or complete tasks related to documents. They play a critical role in workflow processes and task management.

---

## Types of Reviewers

| Type | Description |
|------|-------------|
| :material-account-check: **Workflow Reviewers** | Assigned to workflow steps for approval/rejection |
| :material-clipboard-account: **Task Assignees** | Assigned to individual document tasks |
| :material-certificate: **ISO Reviewers** | Assigned to periodic ISO compliance reviews |
| :material-shield-account: **SOX Reviewers** | Assigned to SOX compliance reviews |

---

## Workflow Reviewers

Workflow reviewers are defined in workflow definitions and automatically receive tasks when documents enter their workflow step.

### Reviewer Actions

| Action | Result |
|--------|--------|
| :material-check: **Approve** | Document advances to next step |
| :material-close: **Reject** | Document returns to submitter |
| :material-pencil: **Request Changes** | Document held for revision |

---

## Task Assignees

Task assignees receive document-based tasks assigned by other users.

### Common Task Types

| Task Type | Description |
|-----------|-------------|
| :material-pencil: **Edit** | Complete or edit the document |
| :material-eye: **Read** | Review and acknowledge the document |
| :material-comment: **Comment** | Provide feedback on the document |

---

## Assigning Reviewers

### In Workflows

1. Create or edit a workflow definition
2. Add a step
3. Select reviewers for the step
4. Configure approval requirements

### For Individual Tasks

1. Select the document
2. Click **Tools** > **Create Task**
3. Select the assignee
4. Set requirements and due date
5. Click **OK**

---

## Reviewer Notifications

Reviewers receive email notifications when:

- A task is assigned to them
- A reminder is due
- A task becomes overdue
